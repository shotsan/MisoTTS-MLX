# MisoTTS MLX: Ultimate Guide & Debug Framework

Welcome to the definitive guide for running MisoLabs' MisoTTS-bf16 (Sesame architecture) natively on Apple Silicon using the MLX framework. 

Voice cloning with large language models on macOS is notoriously tricky due to hidden hardware constraints and library defaults. This document breaks down the underlying architecture, explains the necessary hacks to get it working, and provides a debug framework to prevent silent crashes.

---

## 🧠 Model Architecture & Tokenizer

MisoTTS is built upon the **Sesame Architecture**, which treats Text-to-Speech as a language modeling problem.

### 1. The Dual-LLaMA Engine
Unlike traditional TTS engines, Sesame utilizes two massive autoregressive transformer models working in tandem:
* **The Backbone (LLaMA-8B):** A massive 8-billion parameter engine that analyzes the text semantics, prosody, and the reference voice embedding. It generates abstract acoustic tokens.
* **The Decoder (LLaMA-300M):** A smaller 300-million parameter model that converts the abstract acoustic tokens generated by the backbone into dense, high-fidelity audio waves.

### 2. The Tokenizer & Codebooks
* **Text Vocabulary:** The model uses a massive `128,256` token vocabulary for understanding text.
* **Audio Vocabulary:** It maps audio into `2,051` distinct acoustic tokens.
* **Audio Codebooks:** It utilizes `32` hierarchical codebooks to layer the audio quality, capturing everything from raw pitch to minute breathing sounds.

### 3. The Auto-Transcriber (Whisper)
When you provide a `.wav` file for voice cloning, MisoTTS uses an integrated `mlx-community/whisper-large-v3-turbo-asr-fp16` model to auto-transcribe the audio before passing it to the Sesame model.

## 🛠️ The MLX Hacks & Workarounds

To successfully run this massive 8B+ parameter model natively on a Mac GPU, several critical system-level hacks and library workarounds were required.

### Hack 1: Bypassing the macOS GPU Wired Memory Limit
By default, macOS aggressively restricts how much unified memory the GPU can allocate (the `iogpu.wired_limit_mb`). When the 16-bit 8B model attempts to load its weights and KV cache, macOS will violently kill the process.
* **The Fix:** You must manually unlock the GPU memory limit in the kernel using `sudo sysctl iogpu.wired_limit_mb=22528` (or higher depending on your total RAM).

### Hack 2: Overriding the Hidden 'af_heart' Default
The `mlx_audio` library contains a hidden python default parameter (`voice="af_heart"`). If you delete the voice parameter from your script in an attempt to force raw voice cloning, Python will stealthily inject `"af_heart"` back in, ruining your clone.
* **The Fix:** You must explicitly inject `"voice": None` into the `tts_kwargs` dictionary to obliterate the fallback and force the engine to use your `.wav` file. *(Note: Sesame's internal `generate` function completely ignores the `voice` parameter if it detects a valid `ref_audio` and `ref_text` pair).*

### Hack 3: The Whisper Misalignment Panic
If Whisper fails to transcribe the audio correctly (e.g., it drops words because of background noise, or it truncates the transcript because it detects repeated looping audio), the generated `ref_text` will be shorter than the `ref_audio`. 
When the Sesame model attempts to map the audio to the text and finds a mismatch, the engine will **panic and hallucinate a generic female voice**.
* **The Fix:** Provide clean, 10-15 second reference audio clips. If Whisper fails to transcribe it perfectly, you must manually inject the exact transcript into the `ref_text` parameter to bypass Whisper entirely.

### Hack 4: KV Cache Deadlocks
If you provide an extremely long reference audio (e.g., 2 minutes) or ask it to generate massive paragraphs of text at once, the 16-bit model's Context Window will overflow. Instead of throwing a clean error immediately, MLX will silently deadlock the GPU cache and hang indefinitely.
* **The Fix:** Truncate all reference audio clips to a maximum of 15 seconds using `ffmpeg`. If you are generating a long speech, split the target text into smaller chunks (e.g., 50 words each) and generate them sequentially.

### Hack 5: Removing the AI Watermark
MisoTTS attempts to inject an imperceptible cryptographic watermark into the generated `.wav` files to identify them as AI-generated. This watermark drastically slows down generation and alters the raw audio output.
* **The Fix:** To permanently disable watermarking, you must patch the MLX generation source code. Open `site-packages/mlx_audio/tts/models/sesame/sesame.py` and modify the `generate_result` function. Find the line `if self._watermarker is not None:` and hardcode it to `if False: # WATERMARKING DISABLED`.

## 🎙️ Code Example: Perfect Voice Cloning

To run a flawless voice clone out of the box, we have provided an `examples/example_voice_clone.py` script. Here is the optimal boilerplate code to bypass all the bugs:

```python
from mlx_generate_source import generate_audio

tts_kwargs = {
    # 1. Your target text (Keep under 50-60 words per generation)
    "text": "I think it's fundamentally important that we become a multi-planetary species.",
    "model": "mlx-community/MisoLabs-MisoTTS-bf16",
    
    # 2. CRITICAL: Override the hidden fallback voice!
    "voice": None, 
    
    # 3. CRITICAL: Point to a 10-15 second truncated reference clip!
    "ref_audio": "./examples/elon_reference.wav",
    
    # 4. Auto-transcribe the reference. (If Whisper fails, manually type the transcript here)
    "ref_text": None,
    
    "output_path": "./",
    "file_prefix": "my_perfect_clone",
    "audio_format": "wav",
    "join_audio": True,
}

generate_audio(**tts_kwargs)
```

## 🚀 Setup & Execution

1. **Unlock GPU Memory:**
   Run this in your terminal before executing anything:
   ```bash
   sudo sysctl iogpu.wired_limit_mb=22528
   ```
2. **Convert Audio Formats:**
   MisoTTS works best with standard `.wav` files. Convert `.m4a` or `.oga` files using:
   ```bash
   ffmpeg -y -i input.m4a -ac 1 -ar 44100 output.wav
   ```
3. **Truncate Audio:**
   Ensure your reference audio is no longer than 15 seconds:
   ```bash
   ffmpeg -y -i full_audio.wav -t 15 -c copy ref_audio.wav
   ```
